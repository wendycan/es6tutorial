一、常量

const
1.基本支持
basic support
const foo = 123;
console.log((foo === 123));

结果为 true。

2.块级作用域
is block-scoped
const bar = 123;
{ const bar = 456; }
console.log(bar === 123);

结果为 true。

3.重复定义一个 const 会报错
redefining a const is an error
const baz = 1;
baz = 2;
4.暂时性死区
temporal dead zone
var passed = (function(){
 qux = 2;
}())
const qux = 3;
console.log(qux);

会报错，因为 qux 处于暂时性死区。

5.严格模式的基本支持
basic support (strict mode)
"use strict";
const foo = 123;
console.log((foo === 123));

结果为 true。

6.严格模式的块级作用域
is block-scoped (strict mode)
'use strict';
const bar = 123;
{ const bar = 456; }
console.log(bar === 123);

结果为 true。

7.严格模式重复定义一个 const。
redefining a const (strict mode)
'use strict';
const baz = 1;
try {
  Function("'use strict'; const foo = 1; foo = 2;")();
} catch(e) {
  console.log(true);
}

结果为 true。

8.严格模式的暂时性死区。
temporal dead zone (strict mode)
'use strict';
var passed = (function(){ try { qux; } catch(e) { return true; }}());
const qux = 456;
console.log(passed);

二、let

let

ES6 新增了 let 命令，用来声明变量。它的用法与 var 类似，但有以下几个区别。

1.基本支持
basic support
let foo = 123;
console.log((foo === 123));

结果为 true。

2.块级作用域
is block-scoped
let bar = 123;
{ let bar = 456; }
console.log(bar === 123);

结果为 true。

3.for-loop 声明作用域
for-loop statement scope
let baz = 1;
for(let baz = 0; false; false) {}
console.log(baz === 1);

结果为 true。

4.暂时性死区
temporal dead zone
var passed = (function(){ try {  qux; } catch(e) { return true; }}());
let qux = 456;
console.log(passed);
5.for-loop 迭代器作用域
for-loop iteration scope
let scopes = [];
for(let i = 0; i < 2; i++) {
 scopes.push(function(){ return i; });
}
let passed = (scopes[0]() === 0 && scopes[1]() === 1);
scopes = [];
for(let i in { a:1, b:1 }) {
 scopes.push(function(){ return i; });
}
passed = passed && (scopes[0]() === "a" && scopes[1]() === "b");
console.log(passed);

结果为 true。

6.基本支持（严格模式）
basic support (strict mode)
'use strict';

console.log((foo === 123));
